## 💻블로그를 제작하다.

<br />

### 개요

개발을 공부하다 보니 개발일지, 공부한 내용같은 것을 정리할 나만의 블로그의 필요성을 느꼈다.

velog나 티스토리 등의 서비스를 이용해도 상관은 없었겠지만,

좀 더 커스터마이징이 용이한 웹사이트를 만들고 싶었기에 이것저것 찾아보다 JAMStack이란 것을 알게 되었다.

<br />

### JAMStack?

`JAM` = `J`avaScript + `A`PI + `M`arkup

앱을 미리 빌드해 Static Page를 딜리버리하는 방식으로 서비스한다.

Markdown 문서를 작성하는 방식으로 Post, Page를 관리한다.

사람들이 많이 사용하는 기술이 유망한 기술이라고 믿기에 가장 유명한 Jekyll을 고려해봤으나,

살아생전 접해보지 못한 Ruby 언어를 사용하고 있어 만지려면 배보다 배꼽이 더 커질 것 같더라.

React를 사용하는 프레임워크가 뭐가 있을까 찾아보니 아니나다를까 역시 있었다.

<br />

### Gatsby

Gatsby라는 녀석은 React 기반의 정적 사이트 생성 프레임워크이다.

React는 나에겐 엄청난 장점이었다.

Starter 템플릿들을 이용해 쉽게 시작할 수 있던 것도 좋았고, 다양한 Plugin 생태계도 마음에 들었다.

그리고 공식문서가 너무 잘되어있었다. 웬만한 삽질은 공식문서를 보면서 해결한 것 같다.

<br />

### Netlify

Gatsby는 Netlify를 통한 배포를 적극 권장하고 있다.

Netlify는 Github에 작업한 걸 `Push`만 해도 자동으로 배포해주는 어마어마한 시스템이었다...

그리고 HTTPS, DNS 등 유용한 기능도 많았다.
